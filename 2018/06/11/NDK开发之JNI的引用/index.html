NDK开发之JNI的引用

发表于 2018-06-11 | 分类于 NDK | |

JNI中的引用意在告知虚拟机何时回收一个JNI变量

JNI引用变量分为局部引用和全局引用

局部引用

局部引用，通过DeletLocalRef手动释放对象

原因

  • 访问一个很大的Java对象，使用之后还用进行复杂的耗时操作
  • 创建了大量的局部引用，占用了太多的内存，而且这些局部引用跟后面的操作没有关联性
